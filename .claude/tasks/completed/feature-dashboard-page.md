# タスク: マイページ（/dashboard）作成

## 概要

ログインユーザーの活動拠点となるマイページを作成する。

## 背景

- キャラクター作成画面（`/characters/new`）へのUI導線が存在しない
- 「自分のキャラクターを管理する」ページが欠けている
- `/characters` は「公開キャラ一覧」であり目的が異なる

## 要件

### 機能要件

- [x] `/dashboard` ページを作成
  - [x] 自分のキャラクター一覧を表示（`character.listMine` API）
  - [x] 「新規キャラクター作成」ボタン → `/characters/new`
  - [x] 空状態表示（キャラがない場合のCTA）
  - [x] 各キャラクターから詳細ページへのリンク
- [x] 未ログイン時 → `/login` へリダイレクト

### 非機能要件

- [x] ローディング・エラー状態の表示

## 設計

### 影響範囲

| パッケージ | ファイル | 変更内容 |
| ---------- | -------- | -------- |
| `web` | `src/routes/dashboard.tsx` | 新規作成 |

### 使用API（既存）

- `character.listMine` - 自分のキャラクター一覧取得

### 画面構成

```
/dashboard
├── ヘッダー: 「マイページ」
└── セクション: キャラクター
    ├── タイトル + 「新規作成」ボタン
    ├── キャラクターカード一覧（グリッド）
    └── 空状態: 「まだキャラクターがいません」+ CTA
```

## 実装手順

1. [x] `web/src/routes/dashboard.tsx` を作成
2. [x] `pnpm lint -- --fix && pnpm typecheck`
3. [x] E2Eテスト作成（`e2e/dashboard.spec.ts`）

## 完了条件

- [x] 未ログイン時は `/login` にリダイレクト
- [x] lint/typecheck パス
