# ゲーム概要

## コンセプト

**「TRPGセッションを仕込んで、AIに演じさせて、物語を収穫する」ゲーム**

### 基本仕様

| 項目        | 内容                                   |
| ----------- | -------------------------------------- |
| プレイ形式  | ソロ＋他者キャラ借用                   |
| セッション  | 完全自動生成（プレイヤーは結果を読む） |
| 1セッション | 1ダンジョン探索                        |
| テキスト量  | 3,000〜5,000字程度                     |
| 世界観      | ダークファンタジー、オリジナル         |
| 設計思想    | 数値ではなくナラティブ/ロア中心        |

### プレイヤーの介入ポイント

```
1. キャラ作成
2. パーティ編成（誰を借りるか）
3. ダンジョン選択
4. 行動指針の設定
5. （生成ボタンを押す）
6. 結果を読む・共有する
```

### プレイヤーの楽しみ

- キャラを育て、個性を磨く
- 「この組み合わせで潜らせたら面白そう」という編成の妙
- 生成されたリプレイを読む／共有する
- 他人のキャラとの予想外の化学反応
- コミュニティで世界観を共創する

# キャラクターシステム

## 設計思想

数値ではなくタグ・フレーズ・エピソードでキャラを表現する。

| 従来のTRPG        | このゲーム                               |
| ----------------- | ---------------------------------------- |
| STR: 16           | 「鍛冶屋の息子、鉄塊を軽々と持ち上げる」 |
| HP: 30/45         | 「左腕に包帯、まだ古傷が疼く」           |
| スキル：火魔法Lv3 | 「師匠から継いだ、不完全な炎の秘術」     |

## キャラクター作成フロー

### STEP 1: 種を撒く（ランダム生成）

システムが3〜5つの「断片」を提示。

```
例：
・「灰の中から拾われた子供だった」
・「左目が見えない」
・「誰かを探している」
・「刃物を持つと手が震える」
```

### STEP 2: 芽を選ぶ（プレイヤー選択）

気に入った断片を2〜3個選ぶ。または「全部振り直す」。

### STEP 3: 育てる（AI補助で肉付け）

AIが選んだ断片を繋げて「人物像」を提案。

```
「灰の中から拾われ、育ての親に剣を教わった。
 だが、ある日その親を自らの手で——Loss
 以来、刃物を握ると手が震える。
 今は誰かを探している。贖罪のために。」

→ プレイヤーが承認 or 修正指示 or 再生成
```

### STEP 4: 名付ける

名前を自分で入力、またはAI提案から選ぶ。

### STEP 5: 行動指針を設定

場面ごとの行動傾向を選択肢から選ぶ or 自由記述。

## 断片カテゴリ

| カテゴリ | 役割                   | 生成時の出現 |
| -------- | ---------------------- | ------------ |
| 出自     | どこから来たか         | 必ず1つ      |
| 喪失     | 何を失ったか           | 必ず1つ      |
| 刻印     | 身体的・外見的特徴     | 必ず1つ      |
| 業       | 抱えている罪/呪い/運命 | 0〜1つ       |
| 探求     | 何を求めているか       | 0〜1つ       |
| 癖/性向  | 振る舞いの特徴         | 0〜1つ       |

### 断片の具体例

#### 【出自】

- 灰燼の街の生き残り
- 処刑人の家系に生まれた
- 神殿に捨てられていた子
- 傭兵団で育った孤児
- 没落貴族の末裔
- 魔女に育てられた
- 奴隷市場から逃げ出した
- 死体の山から目覚めた（記憶なし）
- 国境の森に住む民の出
- 異端審問で家族を失った村の子

#### 【喪失】

- かつての仲間を見殺しにした
- 故郷はもう地図にない
- 愛した人を自らの手で葬った
- 名前を奪われた（本名を知らない）
- 神の声が聞こえなくなった
- 右腕の感覚がない
- 笑い方を忘れた
- 帰る場所を燃やしてきた
- 信じていた主君に裏切られた
- 大切な誰かの顔を思い出せない

#### 【刻印】

- 左目が濁った灰色
- 首筋に縄の痕
- 背中一面の鞭傷
- 白髪（若くして）
- 片耳が欠けている
- 焼け焦げた指先
- 瞳孔が縦に裂けている
- 声が枯れて囁くようにしか出ない
- 異様に長い影を引きずる
- 血の匂いを纏っている（本人は気づかない）

#### 【業】

- 殺した数だけ夜に夢を見る
- 嘘をつくと体が痛む呪い
- 月夜に正気を失う
- 死者の声が聞こえる
- 触れたものを腐らせてしまう時がある
- 誰かに命を狙われている
- 人を信じると裏切られる運命
- 自分の死の日を知っている
- 「還りなさい」という声が頭から消えない
- 生きている実感がない

#### 【探求】

- 奪われた妹を探している
- 自分が何者か知りたい
- ある人物を殺すために生きている
- 呪いを解く方法を求めている
- 罪を贖える死に場所を探している
- 禁忌の知識を追い求めている
- かつて交わした約束を果たしに
- 「帰れ」と言った師匠の真意を知りたい
- 世界の果てにあるという故郷を目指している
- 人間に戻る方法を探している

#### 【癖/性向】

- 危険なほど他人を信じやすい
- 朝日を見ると涙が出る
- 寝ている人を起こせない
- 嘘を見抜くが、それを言わない
- 戦闘中だけ饒舌になる
- 子供を見ると目を逸らす
- 酒を飲まないと眠れない
- 約束を絶対に破れない
- 傷ついた者を放っておけない
- 必ず逃げ道を確認する

## 行動指針

AIがセッション中に「このキャラならどう動くか」判断するための軸。

### 指針を設定する場面（4つ）

#### ① 危険を前にしたとき

- 迷わず前に出る
- 慎重に状況を見極める
- 仲間を下がらせ、殿を務める
- 逃げ道を探す
- 話し合いを試みる
- （自由記述）

#### ② 仲間が窮地に陥ったとき

- 何を犠牲にしても助ける
- 助けたいが、無謀はしない
- 冷静に最善手を探す
- 自分の身を優先する（生き延びてこそ）
- 怒りで我を忘れる
- （自由記述）

#### ③ 道徳的選択を迫られたとき

- 正しいと信じる道を選ぶ
- 現実的な利を取る
- 弱い者の側に立つ
- 誰かに判断を委ねる
- 選ばない（両方壊す/逃げる）
- （自由記述）

#### ④ 未知のものに遭遇したとき

- 好奇心が恐怖に勝つ
- 警戒し、距離を取る
- 知識で対処しようとする
- とりあえず触ってみる
- 仲間に任せる
- （自由記述）

## 他者キャラ借用ルール

### ハイブリッド案（推奨）

**基本ルール：**

- 借用は自由、許可不要
- ただし「死亡」「重大な変化」は起きない（致命傷で撤退、など）

**反映されるもの：**

- 「○○と共に△△ダンジョンを踏破した」という履歴
- 軽微な傷、小さなエピソード
- 他キャラとの関係性フラグ（因縁/信頼/借り など）

**反映されないもの：**

- 死亡、永続的な欠損
- キャラの信念を覆すような変化

**公開設定オプション：**

| 設定                | 説明                                 |
| ------------------- | ------------------------------------ |
| `all`（全開放）     | 死亡含め全反映OK                     |
| `safe`（安全）      | ハイブリッドルール適用（デフォルト） |
| `private`（非公開） | 借用不可、身内のみ                   |

# ダンジョンシステム

## 設計思想

ダンジョンは**「キャラの傷を抉り、問いに答えを迫る場所」**として設計する。

単なる「モンスターがいる洞窟」ではなく、物語の触媒として機能する。

## ダンジョンの構造要素

### 基本情報

```yaml
name: "忘却の聖堂"
alias: "神が目を逸らした場所" # 異名
layer_count: 3
recommended_party: "2〜4人"
difficulty_tone: "重い" # 軽い / 普通 / 重い / 絶望的
```

### ロア（3つの時間軸で語る）

```yaml
lore:
  past: |
    かつて、この地には癒しの聖者を祀る聖堂があった。
    巡礼者が絶えず、奇跡すら起きたという。

  fall: |
    ある夜、聖堂の司祭が禁忌を犯した。
    死んだ娘を蘇らせようとして——
    聖堂は一夜で沈黙し、以来、誰も帰ってこない。

  now: |
    今も夜ごと、聖堂の鐘が鳴るのを聞いたという者がいる。
    「帰っておいで」と呼ぶ声とともに。
```

### 雰囲気タグ

```yaml
tags:
  - "#朽ちた神聖"
  - "#悔恨"
  - "#帰れない者たち"
  - "#静寂の恐怖"
  - "#愛ゆえの罪"
```

**用途：**

- AI生成時のトーン制御
- ダンジョン検索/フィルタリング
- キャラとの「共鳴」判定

### 試練タイプ

```yaml
trial_types:
  - "道徳選択" # 正解のない問い
  - "内面対峙" # キャラの過去/傷に向き合う
  - "交渉" # 敵対者/存在との対話
```

**試練タイプ一覧：**

- 戦闘 - 明確な敵との対峙
- 探索 - 隠された道、秘密の発見
- 謎解き - 仕掛け、暗号、象徴の解読
- 道徳選択 - 正解のない問い
- 内面対峙 - キャラの過去/傷に向き合う
- 生存 - 資源管理、時間制限、脱出
- 交渉 - 敵対者/存在との対話

### 層（レイヤー）構造

```yaml
layers:
  - name: "外縁 - 沈黙の参道"
    atmosphere: "石畳を覆う苔、倒れた巡礼者の荷物が散乱"
    possible_events:
      - "過去の巡礼者の幻影が道を示す"
      - "供物台に何かを置くと、道が開く"
      - "仲間の一人が「懐かしい」と呟く（なぜ？）"

  - name: "礼拝堂 - 祈りの残響"
    atmosphere: "壊れたステンドグラス、祭壇に残る黒い染み"
    possible_events:
      - "聖歌が聞こえる。歌っているのは誰？"
      - "祭壇の前で、最も『罪』を持つ者が膝をつく"
      - "告解室から声がする"

  - name: "地下墓所 - 還らずの底"
    atmosphere: "無数の棺、中央に一つだけ開いている"
    possible_events:
      - "司祭の亡霊との対峙"
      - "『娘』が現れる。敵か、救うべき者か"
      - "誰かが『ここに残る』選択を迫られる"
```

### 核心（コア）

```yaml
core:
  nature: "選択" # 選択 / 対決 / 発見 / 喪失 / 解放

  description: |
    司祭の亡霊は問う。
    「私は間違っていたのか？　愛する者を取り戻そうとして」

    『娘』は泣いている。
    死者として在り続けることに疲れ果てて。

  possible_outcomes:
    - "娘を解放する → 司祭は消え、聖堂は静まる"
    - "司祭を肯定する → 娘は永遠に囚われ、代わりに何かを得る"
    - "両者を拒絶する → 聖堂ごと崩壊、脱出戦"
    - "誰かが娘の代わりに残る → 究極の犠牲"
```

### 共鳴トリガー

キャラの断片とダンジョンが「共鳴」すると、特別なイベントが発生。

```yaml
resonance:
  - fragment_type: "loss"
    keywords: ["愛した人", "自らの手で", "家族"]
    effect: |
      『娘』がこのキャラに特に反応する。
      「あなたも、誰かを……？」

  - fragment_type: "sin"
    keywords: ["夢", "罪", "贖い"]
    effect: |
      司祭の亡霊がこのキャラを「同類」と見なす。
      特別な対話ルートが開く。

  - fragment_type: "quest"
    keywords: ["死に場所", "贖い", "終わり"]
    effect: |
      「ここに残る」選択肢がこのキャラに提示される。
```

**設計意図：** 同じダンジョンでも、連れていくキャラによって展開が変わる。

# セッション生成ロジック

## 生成パイプライン全体像

```
[入力]
  ↓
[共鳴スキャン] → triggered_events
  ↓
[プロット生成] → structure (YAML)
  ↓
[シーン分割] → scenes[]
  ↓
[シーンごと本文生成] → scene_texts[]
  ↓
[結合・整形] → final_replay
  ↓
[履歴更新] → updated_character_sheets
  ↓
[出力]
```

## 前処理：共鳴スキャン

パーティ全員の断片をダンジョンの共鳴トリガーと照合。

```python
# 疑似コード
triggered_events = []

for character in party:
    for resonance in dungeon.resonance:
        if resonance.fragment_type in character.fragments:
            fragment_text = character.fragments[resonance.fragment_type]
            if any(kw in fragment_text for kw in resonance.keywords):
                triggered_events.append({
                    "character": character.name,
                    "event": resonance.effect,
                    "priority": "high"
                })
```

**出力例：**

```yaml
triggered_events:
  - character: "灰村のセド"
    event: "『娘』がこのキャラに特に反応する"
    priority: high

  - character: "灰村のセド"
    event: "「ここに残る」選択肢が提示される"
    priority: high
```

## 構成生成：プロット骨子

AIに「物語の設計図」を先に作らせる。

### セッション構成テンプレート

```yaml
structure:
  # === 起：導入 ===
  opening:
    scene: "ダンジョン入口での状況描写"
    party_dynamic: "キャラ同士の初期関係性を示す会話"
    hook: "なぜ今ここに来たのかの動機確認"

  # === 承：探索（層ごと） ===
  exploration:
    - layer: 1
      main_event: "選択 or 発見 or 遭遇"
      character_focus: "誰にスポットを当てるか"
      tension: "low → medium"

    - layer: 2
      main_event: "..."
      character_focus: "..."
      tension: "medium → high"
      triggered_resonance: "（あれば）共鳴イベント発火"

  # === 転：核心 ===
  climax:
    confrontation: "核心での対峙内容"
    choice_bearer: "誰が選択を迫られるか"
    resonance_payoff: "共鳴イベントの回収"

  # === 結：帰還 ===
  resolution:
    outcome: "どの結末を迎えたか"
    cost: "何を得て、何を失ったか"
    changed: "誰がどう変わったか"
```

```markdown

{triggered_eventsのリスト}

- 共鳴イベントは必ず物語に組み込む

- 結末は possible_outcomes のいずれかをベースにする
- 「借用：safe」のキャラは死亡・永続損傷させない
```

## 結末決定のロジック

### 決定要因

```yaml
outcome_selection:
  factors:
    - party_composition: "パーティ内に『贖罪』を求めるキャラがいるか"
    - resonance_events: "どの共鳴イベントが発火したか"
    - directive_alignment: "核心の選択と行動指針の相性"
```

### 最終選択ロジック

```python
# 疑似コード
def select_outcome(party, dungeon, triggered_events):
    scores = {}

    for outcome in dungeon.core.possible_outcomes:
        score = base_score(outcome)

        # 共鳴イベントによる加点
        for event in triggered_events:
            if event.relates_to(outcome):
                score += 0.3

        # 行動指針との相性
        for char in party:
            score += alignment_score(char.directives, outcome)

        scores[outcome] = score

    # 最高スコア＋ランダム揺らぎ
    return weighted_random_choice(scores)
```

# リプレイ出力形式

## 形式の選択

**推奨：ハイブリッド形式**

小説形式ベース＋場面転換を明示。

- 地の文：情景描写、心理描写
- 台詞：キャラクターの発言
- 場面区切り：シーン転換を視覚的に
- 判定は見せない：ナラティブ中心なので数値を出さない


# ユーザー投稿システム

## 投稿できるもの

| 投稿物       | 難易度 | 審査                 | 主な用途               |
| ------------ | ------ | -------------------- | ---------------------- |
| キャラクター | 低     | なし（自動公開）     | 借用、パーティ編成     |
| ロア断片     | 低     | AI自動               | 世界観拡張、素材プール |
| ダンジョン   | 中〜高 | AI＋任意コミュニティ | セッションの舞台       |
| ギルド       | 中     | なし（自動作成）     | コミュニティ形成       |

## キャラクター投稿

### フロー

```
[キャラ作成]
    ↓
[マイキャラとして保存]
    ↓
[公開設定を選択]
    ├→「非公開」── 自分だけ使える
    ├→「借用可・安全」── 他者が借用可、死亡なし
    └→「借用可・全開放」── 死亡含め全反映OK
    ↓
[公開] → コミュニティで発見可能に
```

## ダンジョン投稿

### 投稿フロー

```
STEP 1: 骨子入力
  ・名前、異名
  ・コンセプト（一言で何を描きたいか）
  ・雰囲気タグ選択（3〜5個）
  ・試練タイプ選択（2〜3個）
  ・層数（2〜4）
    ↓
STEP 2: AI補助でロア生成
  入力したコンセプトを元に、AIが提案：
  ・past / fall / now のロア案
  ・各層の雰囲気と候補イベント
  ・核心の選択肢案
  → ユーザーが採用/修正/再生成
    ↓
STEP 3: 詳細調整
  ・各層のイベントを編集
  ・共鳴トリガーを設定（任意）
  ・核心の結末パターンを確定
    ↓
STEP 4: テストプレイ（任意）
  自分のキャラでソロ生成してみる
  → 問題があれば修正
    ↓
STEP 5: 投稿申請
  ・規約確認
  ・タグ最終確認
  ・難易度自己申告
    ↓
STEP 6: 審査
  ・AI自動チェック
  ・コミュニティレビュー（任意）
  ・問題なければ公開
```

### AI自動チェック項目

```yaml
auto_review:
  checks:
    - name: "構造完全性"
      rule: "全ての必須フィールドが埋まっているか"

    - name: "トーン一貫性"
      rule: "ロアと雰囲気タグが矛盾していないか"

    - name: "結末多様性"
      rule: "possible_outcomes が2つ以上あるか"

    - name: "禁止コンテンツ"
      rule: "差別的/性的/過度に暴力的な内容がないか"

    - name: "著作権"
      rule: "既存作品の固有名詞をそのまま使っていないか"

  result:
    - pass → 公開
    - warning → 修正推奨（公開は可能）
    - reject → 修正必須
```

- 組織、勢力の名前と概要
- アイテム、遺物の伝説
- 暦、祭事、風習

# 世界観設定

## 設計思想

**「骨格は固める、肉付けはコミュニティに委ねる」**

**確定させるもの：**

- 世界の「状態」（なぜダークなのか）
- トーンの制約（何を描き、何を描かないか）
- 基本ルール（魔法、死、神など）

**委ねるもの：**

- 具体的な地名、国家、組織の詳細
- 個別の伝承、歴史的事件
- ダンジョンの由来

## 世界の概要

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  灰 暦 の 世 界
  ─ The World of Ashen Calendar ─

  "神々は去り、英雄は朽ち、
   それでも人は歩き続ける"

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**一言で言うと：**
「かつて栄光があった世界の、長い黄昏」

- 神話の時代は終わった
- 大いなる破局が過去にあった
- 今は残骸の中で生きる時代
- 希望は微かだが、ゼロではない

## 三つの時代

### 第一の時代：神座の時代（はるか昔）

**状態：** 終わった

```
神々が地上を歩き、人と言葉を交わした時代。
奇跡は日常であり、死すら絶対ではなかった。
```

**残されたもの：**

- 朽ちた神殿、聖遺物
- 「神の言葉」を記した碑文（ほとんど解読不能）
- 神に仕えた存在の残滓（天使でも悪魔でもない「何か」）

**謎：** なぜ神々は去ったのか、誰も知らない

### 第二の時代：王冠の時代（数百年前まで）

**状態：** 崩壊した

```
神々なき世界で、人は自らの王国を築いた。
魔術は体系化され、文明は栄えた。
七つの大国が覇を競い、英雄譚が生まれた時代。
```

**残されたもの：**

- 廃墟となった城塞、都市
- 失われた魔術の断片
- 「英雄」の墓所（多くがダンジョン化）
- 没落貴族、王家の末裔を名乗る者たち

**崩壊：**
「大禍（たいか）」と呼ばれる破局が訪れた。
何が起きたかは諸説ある。

- 魔術の暴走
- 神々の呪い
- 地の底から這い出た何か
- 人の業が臨界を超えた

結果だけが残った。王国は崩れ、人口は激減し、世界は灰に覆われた。

### 第三の時代：灰暦の時代（現在）

**状態：** 現在

```
大禍から約200年。
世界は少しずつ、瓦礫の中から立ち上がろうとしている。

大国はない。小さな街や集落が点在し、
その間を荒野と廃墟と「危険なもの」が埋めている。
```

**特徴：**

- 旅は危険。街の外は基本的に敵地
- 過去の遺産（ダンジョン）には富と死がある
- 魔術は失われかけているが、断片は残る
- 新しい秩序を作ろうとする者たちがいる
- 古い秩序にしがみつく者たちもいる

## ダンジョンの存在理由

灰暦の世界では、**「強い想念を残した場所」**が時間とともに変質し、ダンジョンと化す。

### ダンジョンの起源タイプ

| タイプ         | 説明                                     | 特徴                                         |
| -------------- | ---------------------------------------- | -------------------------------------------- |
| 神座の遺構     | 神々が去った後に残された聖域、祭壇、神殿 | 神聖さと禁忌が入り混じる。試練を与える傾向   |
| 王冠の廃墟     | 崩壊した城、都市、研究施設               | 人の欲望と栄光の残滓。財宝と罠が多い         |
| 大禍の傷痕     | 大禍で特に被害を受けた場所               | 歪み、汚染、理解不能な現象。危険度高         |
| 想念の結晶     | 強い感情や執念が凝縮した場所             | 悲劇、憎悪、愛、後悔などがテーマになる       |
| 生きている場所 | 意志を持ってしまった空間                 | ダンジョン自体が存在であり、対話可能な場合も |

**共通の特徴：**
全てのダンジョンに共通するのは、「何かを抱えている」ということ。
過去の記憶、誰かの想い、解かれていない問い。
だから探索者は、宝を求めるだけでなく、ダンジョンの「核心」に触れることになる。

## 探索者が存在する理由

### 社会的位置づけ

```
正業ではないが、必要とされている。
街は探索者を「必要悪」として見ている。
危険な場所から資源を持ち帰る者。
厄介事を解決する者。
そして、帰ってこないことも多い者。
```

### 動機

- **金と資源** - 廃墟には価値あるものが眠る
- **知識** - 失われた魔術、歴史の真実を求めて
- **贖罪** - 死に場所を探して、あるいは何かを取り戻すために
- **使命** - 誰かに頼まれた、あるいは導かれた
- **逃避** - 街にいられない理由がある
- **純粋な好奇心** - 世界の秘密を知りたい

### 組織

統一された「冒険者ギルド」のようなものは存在しない。
各地に小さな互助組織、酒場のネットワーク、傭兵団、魔術結社などが点在し、
探索者はそれらに属したり、独立で動いたりする。

## 魔術と超自然のルール

### 現在の状態

```
魔術は「失われかけている技術」である。
神座の時代の奇跡は再現不可能。
王冠の時代の体系的魔術も、大部分が散逸した。

今残っているのは：
- 断片的な術（特定の効果だけを起こす）
- 師から弟子へ口伝される秘術
- 遺物に封じられた力
- 生まれつきの「異能」
```

### 制約

| 制約       | 内容                                                                                     |
| ---------- | ---------------------------------------------------------------------------------------- |
| 代償の法則 | 魔術には必ず代償がある。力を引き出すほど、何かを支払う。体力、寿命、記憶、感情、人間性—— |
| 不完全性   | 完璧に制御できる魔術は存在しない。どこかに綻び、予想外の副作用、限界がある。             |
| 個人性     | 同じ術でも使い手によって形が異なる。師匠の炎と弟子の炎は同じではない。                   |

### 魔術にできないこと

- 死者を完全に蘇らせる（歪んだ形でなら可能、だが...）
- 時間を巻き戻す
- 神を呼ぶ
- 世界の法則そのものを書き換える

### 物語上の意味

魔術は「数値的な強さ」ではなく、「何を犠牲にしてきたか」「どんな形で現れるか」でキャラクターの物語を語る要素となる。

## 神と信仰

### 去った神々

**状態：** 不在

```
神々はかつて確かに存在した。その痕跡は世界中にある。
だが今、祈りに応える者はいない。
```

**諸説：**

- 神々は人に失望して去った
- 神々は大禍で死んだ
- 神々は眠っている
- 神々は最初から人が作った幻想だった
- 神々はまだ見ているが、もう干渉しないと決めた

**真実：** 誰も知らない。これはコミュニティが探求するテーマ。

### 現在の信仰

| タイプ          | 説明                                                                                                     |
| --------------- | -------------------------------------------------------------------------------------------------------- |
| 遺された教団    | 去った神々を今も信じ、いつか帰還すると待つ者たち。儀式を続け、教えを守る。奇跡は起きないが、信じている。 |
| 新しい信仰      | 大禍を生き延びた「何か」を神として崇める者たち。あるいは、人の中に神性を見出そうとする思想。             |
| 無神論/現実主義 | 神はいない、いたとしても関係ない、と割り切る者たち。灰暦の世界では珍しくない態度。                       |

## 死と魂

### 基本ルール

```
この世界では、死は基本的に終わりである。
簡単には覆せない。
```

### ただし

```
強い想念は残ることがある。

- 亡霊、残留思念として
- ダンジョンの一部として
- 呪いや祝福として

完全な復活は不可能だが、
「会話する」「未練を解く」「送る」ことは
物語の中で起こりうる。
```

### ゲームプレイへの影響

- キャラクターの死は重い（特に「全開放」設定の場合）
- 死んだキャラは「伝説」として残る可能性がある
- 死者との再会は、ダンジョンの核心として起こりうる

## トーンガイドライン

### 核心の雰囲気

- 希望はあるが、容易ではない
- 美しさと残酷さが共存する
- 善悪は単純ではない
- 過去の重みが現在にのしかかる
- それでも人は選択し、歩き続ける

### 推奨される表現

- 喪失と再生の物語
- 道徳的に曖昧な選択
- 小さな希望、微かな光
- キャラクターの内面の葛藤
- 世界の謎への好奇心
- 痛みを伴う成長
- 予想外の優しさ

### 避けるべき表現

- 単純な勧善懲悪
- 簡単に手に入るハッピーエンド
- 意味のない残虐描写
- 性的な内容
- 現実の差別を再生産する表現
- 完全な絶望（救いの可能性がゼロ）

### 美学キーワード

灰、煤、錆 / 黄昏、薄明、月光 / 廃墟、遺構、残骸 / 傷痕、包帯、古傷 / 沈黙、囁き、残響 / 記憶、忘却、面影

## 初期ロア断片（公式設定）

### 地名

**灰燼の街**

```
かつて鍛冶で栄えた街。
「灰の夜」と呼ばれる大火で一夜にして滅びた。
原因は諸説ある——事故、放火、呪い、あるいは内側から燃えた。
今は廃墟のみが残り、時折、生き残りを名乗る者が現れる。
```

**渡りの海**

```
大陸を東西に分かつ内海。
大禍以前は交易路として栄えたが、今は渡る者も少ない。
海の底には沈んだ都市があるという。
```

**最後の灯台**

```
渡りの海の岬に立つ、まだ灯りが点る灯台。
誰が維持しているのか、誰も知らない。
嵐の夜、灯台を目指した船は必ず岸に着くという。
ただし、乗っていた者が同じ人間かどうかは——
```

### 伝承

**七つの王冠**

```
王冠の時代、七つの大国を治めた七人の王。
それぞれが冠を戴き、それぞれが滅びた。
七つの王冠は今も世界のどこかに眠り、
見つけた者に力を、あるいは破滅を与えるという。
```

**帰らずの鐘**

```
いくつかの古い聖堂には「帰らずの鐘」がある。
鳴らすと、去った者が帰ってくるという。
ただし、帰ってきた者が望んだ姿かどうかは別の話。
```

### 組織

**灰拾い**

```
各地に点在する探索者の緩やかなネットワーク。
統一された組織ではなく、「灰拾い」を名乗る酒場や
宿が各地にあり、情報と仕事を融通し合う。
合言葉は「灰の中に、まだ火種はある」。
```

**記憶の書庫**

```
失われた知識を収集する学者たちの結社。
各地の廃墟から書物、碑文、遺物を回収し、
解読と保存を行っている。
中立を旨とし、知識を求める者には分け隔てなく門を開く。
ただし、「禁書庫」には誰も入れない。
```

### 遺物

**語りの灯**

```
掌に収まる小さなランタン。
火を灯すと、その場所で起きた過去の出来事が
幻影として浮かび上がるという。
ただし、見たいものが見えるとは限らない。
```

**忘却の石**

```
握ると、一つだけ記憶を消せるという黒い石。
消した記憶は二度と戻らない。
それでも求める者は後を絶たない。
```

## 世界設定の「余白」

明示的に「未確定」とする領域。コミュニティが埋めていく。

### 地理

- 大陸の全体図（部分的にのみ確定）
- 各地の街や集落の詳細
- まだ名前のない廃墟、ダンジョン

### 歴史

- 大禍の真の原因
- 七つの王国それぞれの歴史
- 神座の時代の具体的な出来事

### 勢力

- 現存する組織の詳細
- 新しい勢力の興亡
- 対立構造、同盟関係

### 形而上学

- 神々の正体と去った理由
- 魔術の起源
- ダンジョンの最深層には何があるのか

**注：** これらは「答えがない問い」ではなく、「コミュニティが探求し、創造していく余地」である。矛盾する設定が投稿された場合、「諸説ある」「地域によって異なる」として共存させることを基本とする。

# コミュニティ設計

## コミュニティの3本柱

| 柱       | 概要       | 主な機能                                   |
| -------- | ---------- | ------------------------------------------ |
| **発見** | 見つける   | フィード、検索、レコメンド、ランキング     |
| **交流** | つながる   | フォロー、コメント、ギルド、通知           |
| **共創** | 一緒に作る | ロア投稿、ギルド活動、イベント、世界観拡張 |

## 発見システム

### ホームフィード構成

- 最近の冒険（リプレイ）
- 話題のキャラクター
- 新着ダンジョン
- フォロー中の活動

### フィードアルゴリズム

| ソース                     | 重み |
| -------------------------- | ---- |
| フォロー中のユーザーの活動 | 0.3  |
| ギルドメンバーの活動       | 0.25 |
| 人気コンテンツ（全体）     | 0.2  |
| あなたへのおすすめ         | 0.15 |
| 新着                       | 0.1  |

### パーソナライズ要因

- 過去に借用したキャラの傾向
- よく潜るダンジョンのタグ
- よく読むリプレイの結末タイプ
- 所属ギルドの傾向

### 検索・フィルタリング

**キャラクター検索：**

- 断片タイプフィルタ
- タグフィルタ
- 借用設定フィルタ
- ソート（新着/人気/生還率/波乱度）

**ダンジョン検索：**

- 雰囲気タグフィルタ
- 試練タイプフィルタ
- 難易度フィルタ
- ソート（新着/人気/評価）

### レコメンデーション

**パーティ編成時：**

- 相性補完（異なる行動指針を持つキャラ）
- 共鳴期待（ダンジョンの共鳴トリガーに合うキャラ）
- 物語的対比（ドラマが生まれそうな組み合わせ）

**ダンジョン選択時：**

- キャラとの共鳴
- 未踏破ダンジョン
- 物語の続き（前回のテーマを深掘り）

### ランキング・リーダーボード

- 最も読まれたリプレイ
- 最も借用されたキャラ
- 最も挑戦されたダンジョン
- 伝説のセッション（殿堂入り）

## 交流システム

### ソーシャル機能

**フォロー：**

- 新しいキャラ作成
- セッション完了
- ダンジョン投稿
- ロア断片投稿

**リアクション：**

- ♡ 心に残った
- 🔥 熱い展開
- 😢 泣いた
- 🤔 考えさせられた
- ⚔️ 見事な戦い

**コメント：**

- リプレイへのコメント
- キャラクターへのコメント
- ダンジョンへのレビュー
- シーン引用コメント（特定のシーンに対して）
- ネタバレ折りたたみ

### 通知システム

**あなたのキャラ：**

- キャラが借用された
- キャラが登場するリプレイが公開された
- キャラにコメントがついた

**あなたの投稿：**

- ダンジョンがプレイされた
- リプレイにリアクション/コメント
- ロア断片が引用された

**ソーシャル：**

- 新しいフォロワー
- フォロー中の人の新作
- ギルドメンバーの活動

### ギルド内交流

- 掲示板（固定投稿、一般投稿）
- 最近のギルド活動フィード
- メンバー一覧
- ギルド専用チャット（将来）

## 共創システム

### ロア断片の連鎖

誰かが投稿したロア断片を「引用」して、別のロア断片を作ることができる。

**例：**

```
「灰燼の街」（オリジナル）
    ↓ 引用
「灰の夜」（派生：大火の詳細）
    ↓ 引用
「灰燼の鍛冶」（派生：鍛冶師の秘術）
```

### ロアマップ

ロア断片の関係性を視覚的に表示。クリックで詳細表示、新しいロアを追加。

### 世界観イベント

**発見イベント：**

- 新しいダンジョンが発見された
- 期間中、特定のダンジョンに多くの探索者が挑む
- 結果の統計が公式ロアに反映される

**謎解きイベント：**

- 世界観の「謎」をコミュニティで解き明かす
- ロア断片を投稿して仮説を立てる
- 最も支持された説が「有力説」として採用

**共創イベント：**

- テーマを決めてダンジョンやキャラを募集
- 優秀作は「公式採用」

### ギルド間関係

| 関係     | 効果                                               |
| -------- | -------------------------------------------------- |
| 同盟     | 互いのキャラを借用しやすい、合同ダンジョン作成可能 |
| ライバル | ランキングで比較表示、対抗戦イベント参加可能       |
| 対立     | 同じダンジョンで異なる陣営として遭遇、対決イベント |
